MUST-HAVE FOR LAUNCH (Table stakes — users expect these)
=========================================================

F1. PAYMENT PROCESSING (Stripe Connect)
-----------------------------------------
Status:     Not started — no payment code exists anywhere
What:       Full Stripe integration for subscriptions, PPV, tips
Requires:   - Stripe Connect for creator onboarding
            - Stripe Checkout / Payment Intents for purchases
            - Webhook handler (Supabase Edge Function)
            - Billing cycle management (recurring subscriptions)
            - Failed payment retry and grace period
            - Refund handling
            - Invoice/receipt generation
            - SCA/3D Secure support for EU users
Complexity: HIGH (2-3 weeks)
Note:       Adult content has limited payment processor options.
            Stripe allows it with restrictions. CCBill and Segpay
            are industry-standard alternatives if Stripe declines.

F2. SUBSCRIPTION CANCELLATION / MANAGEMENT
--------------------------------------------
Status:     Not started — no unsubscribe UI or logic
What:       Allow subscribers to cancel, and handle expiry gracefully
Requires:   - Cancel button on subscribed creator's profile
            - Confirmation modal
            - Cancel at end of billing period (not immediate)
            - Resubscribe flow
            - Subscription history page for users
Complexity: MEDIUM (3-5 days)

F3. TIPPING
-------------
Status:     Schema exists (tip_amount on messages), no UI
What:       Allow fans to send tips to creators
Requires:   - Tip button on creator profile and posts
            - Amount selection (preset + custom)
            - Payment processing
            - Tip notification to creator
            - Tip history/leaderboard
Complexity: MEDIUM (3-5 days, assumes payment integration exists)

F4. CONTENT REPORTING + MODERATION
------------------------------------
Status:     "Report" button exists in dropdown, no handler
What:       Allow users to report content and creators to moderate
Requires:   - Report flow (select reason, optional description)
            - reports table in DB
            - Admin moderation queue (or use Supabase dashboard)
            - Auto-hide after N reports threshold
            - Email notification to admins
            - DMCA takedown request handling
Complexity: MEDIUM (1 week)

F5. USER BLOCKING / MUTING
-----------------------------
Status:     "Mute" button exists, no handler
What:       Allow users to block or mute other users
Requires:   - blocks table (blocker_id, blocked_id)
            - Block button on profile and post dropdown
            - Filter blocked users from feeds, search, DMs
            - RLS policies to enforce blocks
            - Mute = hide from feed without blocking
Complexity: MEDIUM (3-5 days)

F6. ACCOUNT DELETION (GDPR REQUIREMENT)
-----------------------------------------
Status:     Not started
What:       Allow users to delete their account and all associated data
Requires:   - Delete account button in settings
            - Confirmation flow (password re-entry)
            - Cascade delete all user data (posts, media, messages, etc.)
            - Delete storage files
            - Cancel active subscriptions
            - Hard delete or anonymize per GDPR requirements
            - 30-day grace period before permanent deletion (optional)
Complexity: MEDIUM (3-5 days)

F7. EMAIL / PASSWORD CHANGE
-----------------------------
Status:     Not started
What:       Allow users to change email and password from settings
Requires:   - Change email flow (verify new email)
            - Change password form (current + new password)
            - Supabase auth.updateUser() integration
Complexity: LOW (1-2 days)

F8. COMMENT REPLIES / THREADING
---------------------------------
Status:     DB has parent_id column, no UI
What:       Allow users to reply to comments, creating threaded discussions
Requires:   - Reply button on each comment
            - Nested comment display (1-2 levels)
            - Reply notification to parent commenter
            - Collapse/expand thread UI
Complexity: MEDIUM (3-5 days)
